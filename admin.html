Breach Response Protocol
01
Detect and Confirm

Identify the breach scope — which tables, which users, what data was exposed. Check Supabase logs and scraper run logs in /logs/.

02
Contain Immediately

Rotate Supabase anon key and service key. Revoke all active sessions. Take the platform offline temporarily if needed.

03
Notify Affected Users

Send breach notification emails to all affected accounts within 72 hours. Describe what data was exposed and what action to take.

04
Patch and Harden

Fix the vulnerability. Review all RLS policies in Supabase. Update API key exposure rules. Force all users to reset passwords.

05
Document and Report

Log the full incident timeline. If student personal data was exposed, report to relevant Jamaican data protection authority within required timeframe.
